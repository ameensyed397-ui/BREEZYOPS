## CRITICAL Issues (2)

### 1. No mobile navigation
`app/(app)/layout.tsx:18` — sidebar is `hidden md:flex`. On phones/tablets (< 768px) the entire app is **unusable** — no hamburger menu, no bottom nav, no drawer. Users cannot navigate between pages.

### 2. Missing database indexes
`lib/db/schema.ts` — zero indexes on any foreign key or frequently-queried column. Every query on `leads`, `jobs`, `invoices`, `appointments`, `deals`, `customers` will do a full table scan. Critical for performance at any data volume.

---

## HIGH Issues (21)

### 3. No `loading.tsx` files anywhere
Zero loading states across all routes. Users see a blank page during every server-side data fetch. No skeletons, no spinners.

### 4. No `error.tsx` files anywhere
Zero error boundaries. A single failed DB query crashes the entire page to Next.js default error UI — no recovery, no retry, no helpful message.

### 5. All actions are toast-only stubs
~20+ button handlers across the app fire `toast.success()` and do nothing. Qualify lead, book appointment, start job, mark complete, send invoice, record payment, create customer — all fake. No actual state mutations.

### 6. No keyboard navigation for drag-and-drop
Kanban boards (`components/pipeline/kanban-board.tsx`) use `@dnd-kit` but no keyboard sensor is configured. Keyboard-only users cannot move cards. WCAG violation.

### 7. No form labels on login inputs
`app/(auth)/login/page.tsx:76-98` — email/phone/OTP inputs have only `placeholder`, no `<Label>` or `aria-label`. Screen readers announce "edit text" with no context.

### 8. No `aria-label` on any list row button
Every list component (leads, customers, jobs, invoices, documents) renders rows as `<button>` with no `aria-label`. Screen readers can't parse nested content.

### 9. No search on leads inbox
`components/leads/lead-inbox.tsx` — unlike every other list, leads has no search/filter input. Users can't find leads by name or phone.

### 10. Customer detail jobs tab is dead
`components/customers/customer-detail-sheet.tsx:85` — only shows `lastJob`. The "+ N more jobs" text is not clickable. No way to see actual job records.

### 11. Settings catalog is read-only
`components/settings/settings-panel.tsx` — "Save" button fires toast but table rows are read-only `<td>` elements. Users can't edit anything.

### 12. No mobile nav = no responsive tables
Settings catalog (6 columns), invoice line items, week-view grid (7 columns) — all break on mobile with no horizontal scroll wrapper.

### 13. Dashboard hardcoded
`app/(app)/page.tsx` — all KPI values, greeting ("Good morning, Asad"), and lead badge count are hardcoded strings.

### 14. SLA timer is static
`components/leads/lead-inbox.tsx` — "20m left" is computed once via `useMemo`, never counts down.

### 15. Kanban loading is fake
`components/pipeline/kanban-board.tsx:40` — `setTimeout(350ms)` always shows skeleton for exactly 350ms regardless of data.

### 16. GST won dialog doesn't persist
`components/pipeline/b2b-board.tsx:55` — `confirmWon()` updates local state only, lost on refresh.

### 17. Week view breaks on mobile
`components/schedule/week-view.tsx:34` — `grid-cols-7` with no responsive override. ~45px per column on phones.

### 18. Schedule board never loads
`components/schedule/schedule-board.tsx:31` — `const [loading] = useState(false)` hardcoded. DayView skeleton never shows.

### 19. Booking form double-submit
`components/schedule/booking-sheet.tsx:175` — "Book appointment" button has no `disabled`/loading state. Double-click creates duplicates.

### 20. Login: no resend code
`app/(auth)/login/page.tsx` — after entering verify stage, no way to request a new OTP.

### 21. Login: no input type validation
Email field accepts any string (no `type="email"`), phone accepts any string (no `type="tel"`).

### 22. InfoCard duplicated 4 times
Same component defined in `customer-detail-sheet.tsx`, `job-detail-sheet.tsx`, `invoice-detail-sheet.tsx`, `document-detail-sheet.tsx`.
